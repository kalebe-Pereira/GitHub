            VINCULANDO O GIT
---------------------------------------------------------------------------------
Para vincular o git, é necessidade cadastrar o E-mail e o nome de usuário, para cadastrar  o E-MAIL, faz da seguinte maneira:

    git config --global user.email "SEU_E-MAIL"

Para cadastrar o nome de USUÁRIO, faz da seguinte forma:

     git config --global user.name "Your Name"

----------------------------------------------------------------------------------

            COPIANDO GITHUBS
----------------------------------------------------------------------------------
Para vincular o GitHub com git o git, será necessário a chave SSH. O SSH faz uma conexão segura e permanente com o pc.

Para gerar a chve SSH, é necessário digitar o seguinte comando no terminal:

    ssh-keygen -t ed25519 -C "email"

Para pegar a chave SSK, precisa-se digitar o seguinte comando no terminal:

    cat ~/.ssh/id_ed25519.pub

Para testar a conexão, utiliza-se o comando:
    ssh -T git@github.com
-----------------------------------------------------------------------------------

Para iniciar os comandos em Git, é necessário adicionar a palavra "git" antes dos comandos.

add -> Serve para adicionar u arquivo para receber modificações do git.
    Exemplo: git add arquivoExemplo.cpp
    Pode-se Utilizar o ponto(.) para selecionar todos os arquivos da pasta para add.

    A -> Atualiza somente os que já estão no staged.
        Exemplo: git add -a

    *.<tipo_de_dado> -> Leva ao stage somete os arquivos do tipo informado.
        Exemplo: git add *.py
reverse - > Serve para retirar um add do arquivo que você adicionou.
    Exemplo: git reverse <nome_do_arquivo>

diff -> Serve para ver as modificações do arquivo no terminal.
    Exemplo: git diff arquivoExemplo.cpp

log -> Serve para ver todos os comit adicionadas.
    Exemplo: git log
    oneline -> Serve para ver toda a informação do commit em uma linha.
        Exemplo: git log --oneline

    all -> Serve para ver todos os commit.
        Exemplo: git log --all

switch -> Serve para voltar em algum commit. É necessário o link do log para isso.
    Exemplo: git switch --datach <link>

restore -> Serve para rvoltar o arquivo do mesmo modo do último add.
    Exemplo: git restore --staged arquivoExemplo.cpp

status -> Verifica os arquivos em add.
    Exemplo: git status

revert -> Serve para reverter algum commit feito (não apaga o commit). É necessário o link do log para isso.
    Exemplo: git revert <link>

commit -> Serve para salvar as verções adicionadas.
    Exemplo: git commit arquivoExemplo.cpp

    A -> Serve para adiciar os arquivos no stage e commitar-los ao mesmo tempo.
        Exemplo: git commit -a -m "Mensagem para a o commit"

    amend - "Atualiza o último commit sem a necessidade de adicionar outro. O amend só funciona no último commit.
        Exemplo: git commit --amend -m "Atualizando o commit anterior"

                    BRANCH
-----------------------------------------------------------------------------------------

As Branchs são como uma liha do tempo como por exemplo, todos os commits até agora fazem parte da branch master (linha do tempo master, ou principal). Ela serve para fazer algumas alterações sem mexer no arquivo principal.

Comandos relacionados a branch:

branch -> Serve para criar uma nova branch.
    Exemplo: git branch <nome_da_branch>

    M -> Serve para modificar o nome da branch. Você deve está dentro dela para modificar.
        Exemplo: git branch -m <novo_nome_da_branch>
        
    branch -> verifica todos os branch feitos.
        Exemplo: git branch

switch -> Acessa a branch. Basta colocar o nome da branch no caminho.
    Exemplo: git switch <nome_da_branch>

Graph -> Uitlizada no log, serve para ver os commit e suas ramificações.
    exemplo: git log --graph 

merge -> Serve para misturar duas branch, caso gostou da alteração e deseja adicionar no principal.
    Exemplo: git merge <nome_da_branch>

    abort -> Para desfazer o merge e desfazer todas as alterações.
        Exemplo: git merge --abort

    quit -> Serve para desfazer o merge.
        Exemplo: git merge --quit

rebase -> serve para adicionar as últimas versões da main para uma branch. Utilizada quando a main é Atualizada antes da branch. Serve para fazer o contrário também, para Atualizar a main com as últimas versões da branch.
    Exemplo: git rebase <nome_da_branch_secundaria>.
    
-----------DIFERENÇA ENTRE REBASE E MERGE ----------
O merge preserva o histórico das branch e a ordem original dos commits, enquando o rebase cria um único histórico linear.
-------------------------------------------------------------------------------------------------------------------

para deletar o branch, pode ser feito de modo abaixo:
    ----- git branch -D <nome_da_branch> -----------
